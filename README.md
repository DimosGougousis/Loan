A working analytics platform for a synthetic Dutch residential-mortgage portfolio. Built end-to-end with **Claude Code** as a demonstration of AI-assisted building inside a regulated financial-services environment.

> ⚠️ **Synthetic data only.** This repository contains no PII and no real loan data. All loan records are generated by a deterministic, version-pinned synthetic generator. GDPR (AVG) is not engaged.

## What it does

Five capabilities, all on a synthetic Dutch mortgage tape:

1. **Quick Scan** — portfolio KPIs at a glance (exposure, weighted PD, current LTV, IFRS 9 stage mix, NHG share, energy-label mix, *aflossingsvrij* share).
2. **Portfolio Analysis** — vintage × arrears heatmap, geographic concentration (PC4 / *gemeente*), LTV distribution, *rentevast* expiry waterfall, energy label × *bouwjaar* mix.
3. **Stress Testing** — four NL-tuned scenarios: DNB house-price shock, HRA phase-out, *rentevast* reset (+200 bps), energy-label transition.
4. **Trend & Anomaly Detection** — vintage trend lines, KS-test drift detection, isolation-forest anomalies with per-loan reasoning.
5. **Loan-Tape Validation** — Pydantic schema + NL-specific cross-field rules (NHG cap, *hypotheekrenteaftrek* eligibility, *Tijdelijke regeling* LTI, BCBS 239 dimensions, AnaCredit conformance).

Plus a **Governance Dashboard** (page 6) that shows where every tape and every AI-assisted build feature is in its pipeline — the platform governs itself.

## Quick start

```bash
git clone <repo>
cd "Synthetic Loan"
uv sync
uv run streamlit run app/streamlit_app.py
```

Generate a fresh synthetic tape:

```bash
uv run python -m loan_tape.generator --seed 42 --n 10000 --out data/samples/synthetic_tape_v1.parquet
```

Run the test + eval suite:

```bash
uv run pytest                  # unit + integration tests
uv run python evals/runner.py  # eval rubrics
```

## Scope boundary — where this platform sits

This is a **post-origination credit-risk portfolio analytics platform**. It is *not* a mortgage origination system. The full Dutch mortgage lifecycle, the upstream platform that produces our input tape, the downstream consumers of our output, and the explicit reasoning for picking this slice over the customer-journey alternative are documented in [`docs/scope-boundary.md`](docs/scope-boundary.md). Read that doc before assessing scope.

## Why this exists

This is a portfolio piece for the **AI-Assisted Builder (Claude / AI Agents) — Credit Risk Portfolio Analysis Platform** role at a Dutch bank. The interesting artifact is not the Streamlit app — it is the **harness around Claude** that kept the build correct and defensible to a 2nd-line reviewer:

- A `CLAUDE.md` pinning the Dutch mortgage schema as single source of truth.
- Seven skills in `.claude/skills/` encoding our build patterns (TDD, validation rules, stress scenarios, SICR triggers, verification).
- Hooks that run tests and lint on every edit, tag AI-authored commits, and block code that violates TDD.
- An eval suite with rubrics for stress scenarios, SICR triggers, and validation.
- A Three Lines of Defense governance model with RACI, change-management gates, and a fail-gracefully playbook.
- A one-page EU AI Act position mapping Articles 9/10/13/14/15 to concrete CI controls.

Read [`docs/build-narrative.md`](docs/build-narrative.md) for the story of how Claude wrote ~70% of this and what kept it correct.

## Regulatory anchors

The schema, rules, and stress scenarios are anchored to:

- **IFRS 9** (staging, ECL)
- **EBA GL/2017/06** (SICR)
- **CRR III / Basel IV** (risk weights)
- **DNB** guidance on AI (2024) and climate stress
- **BCBS 239** (data-quality dimensions)
- **AnaCredit** (ECB Reg 2016/867 — attribute crosswalk in `docs/appendix/anacredit-mapping.md`)
- **NHG / WEW** (Dutch national mortgage guarantee)
- ***Tijdelijke regeling hypothecair krediet*** + **Nibud** norms
- **EU AI Act** (Reg 2024/1689) and **DORA** (Reg 2022/2554)

Full map in [`docs/regulation-map.md`](docs/regulation-map.md).
