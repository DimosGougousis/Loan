# source-command-governance-check

Use this skill when the user asks to run the migrated source command `governance-check`.

## Command Template

# /governance-check

Check the current diff against governance requirements before opening a PR.

## Procedure

1. Compute `git diff --name-only main...HEAD` (or working tree if no branch).
2. Classify changed paths:
   - **Regulated paths** (require 2nd-line): `src/loan_tape/ecl/**`, `src/loan_tape/analytics/stress.py`, `src/loan_tape/validate/rules.py`, `evals/rubrics/**`.
   - **Schema changes**: `src/loan_tape/schema.py`.
   - **Documentation**: `docs/**`.
   - **Harness**: `.Codex/**`, `.github/**`, `pyproject.toml`.

3. For each classification, verify:
   - [ ] Tests updated (`tests/unit/<matching path>/test_*.py` touched).
   - [ ] Eval rubric updated (regulated paths only).
   - [ ] `docs/governance/model-inventory.md` updated (if new quantitative rule).
   - [ ] `docs/regulation-map.md` updated (if new regulation cited).
   - [ ] AI-authored commits all carry `[ai-assisted]` trailer.
   - [ ] No commit > 500 changed lines without re-affirmation.
   - [ ] EU AI Act articles touched flagged in PR description.

4. Print a checklist in PR-ready Markdown:
   ```
   ## Governance checklist (auto-generated by /governance-check)
   - [x] Tests updated for src/loan_tape/ecl/sicr.py
   - [x] eval rubric evals/rubrics/sicr_triggers.md updated
   - [x] Model inventory entry added: climate_transition_trigger v1
   - [x] regulation-map.md updated with DNB climate guidance citation
   - [ ] 2nd-line-approved label (apply after MRM review)
   - [x] [ai-assisted] trailers present on all 4 commits
   - EU AI Act articles touched: Art. 10 (data governance), Art. 15 (accuracy/robustness)
   ```

5. If any required box cannot be ticked, list the missing artifact and exit non-zero.
